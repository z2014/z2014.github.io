webpack

前一阵子时间折腾了好久的webpack，今天就来记录一下之前的收获

webpack是什么

webpack是一款模块加载器兼打包工具，能把各种资源(eg：js，css，img等)都作为模块来处理和使用

配置webpack

每个项目下都要创建一个webpack.config.js文件，他是一个配置项，用来告诉webpack该做哪些事

安装依赖有如下两种方式

  • npm install name –save ，这是发布之后还要依赖的东西
  • npm install name –save-dev，是开发时候依赖的东西

比如，你写 ES6 代码，如果你想编译成 ES5 发布那么 babel 就是devDependencies。
如果你用了 jQuery，由于发布之后还是依赖jQuery，所以是dependencies。

这是webpack打包react的一个简单配置

这是一个基本的webpack配置文件,

  • entry是项目的入口文件，文件支持数组形式，事例中配置是在多页面站点中
  • output是对应输出配置，指输出什么名字的文件存放在什么位置，事例中会生成page1.js和page2.js
  • module模块他会告知webpack每一种文件需要什么加载器来处理

加载 CSS 需要 css-loader 和 style-loader，他们做两件不同的事情，css-loader会遍历 CSS 文件，然后找到 url() 表达式然后处理他们，style-loader 会把原来的 CSS 代码插入页面中的一个 style 标签中,在引入css的时候尽量让每个组件加载自己的css，并且让组件名和 CSS 中的 class 使用一个命名空间

extensions：自动补全识别后缀，是一个数组。如果你想请求一个js文件但是在请求时不带扩展,那么就需要将’.js’添加到数组中。其他文件一样
